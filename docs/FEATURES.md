# TuneCamp Features

## Core Features
-   **Audio-First**: Automatic metadata extraction from MP3, FLAC, OGG, WAV.
-   **Static Generation**: Zero-database HTML generation for portable, fast sites.
-   **Responsive Design**: Mobile-friendly default theme.
-   **Built-in Player**: HTML5 audio player with continuous playback.
-   **Metadata**: YAML-based configuration for artists, releases, and tracks.

## Content Management
-   **Feeds**: Automatic RSS/Atom feed generation.
-   **Podcasts**: Podcast RSS support.
-   **Playlists**: Auto-generated M3U playlists.
-   **Waveforms**: Visual waveform generation for tracks.
-   **Lyrics**: Support for displaying track lyrics.

## Commerce & Access Control
-   **Download Modes**: Free, Pay-what-you-want (Paycurtain), or Protected.
-   **Unlock Codes**: Decentralized download protection using GunDB.
-   **Payment Links**: Integration with PayPal, Stripe, Bandcamp.

## Decentralized / Community
-   **GunDB Integration**: Used for comments, unlock codes, and peer discovery.
-   **Community Directory**: Automatic registration to a public directory of TuneCamp sites.
-   **ActivityPub**: Federation support for following artists, viewing releases, and reading posts via Mastodon/Fediverse.

## Server Mode
-   **Streaming Server**: Personal streaming server for your library.
-   **Web Interface**: Dynamic UI for browsing and playing music.
-   **Admin Studio**: Web-based file browser, track uploads, and management tools.
-   **Multi-User Admin**: Create multiple admin accounts with distinct credentials.
-   **Artist-Restricted Access**: specific admins to artists to restrict their access to only their own content.
-   **Custom Posts**: Microblogging feature for artists to publish news and updates directly to the Fediverse.
-   **Subsonic API**: Full compatibility with Subsonic apps (DSub, Symfonium, etc.) for streaming on the go.
-   **Identity & Backup**: Full portability of server identity and keys for easy migration.
